void uart_send_and_abort_test ( uint32_t  instance)

This tests send async, the transmit status, and the transmit abort. It sends a sting of characters and attempts to abort the transmission in the middle of sending the data. Depending on the size of FIFO, either a minimum of 3 bytes or more will cause the transmit to stop. For FIFO sizes of 8, the transmit will stop at 8 bytes because the FIFO is filled in the send function.

void uart_send_receive_driver_test ( uint32_t  instance)

This tests the back-to-back reception from a terminal program and sends the data back to the terminal (echo) one byte at a time to test the efficiency of the transmit/receive UART drivers.

uart_status_t uart_async_driver_test ( uint32_t  instance)

This test attempts to test the full duplex nature of the UART by transmitting and receiving simultaneously, which can only be achieved using the async functions. This test is a while loop that constantly transmits data to a serial terminal. While transmitting, the user types 10 characters in the terminal window. When the 10th character is typed, the test will output that data among the transmit stream. To end the test, type 'x' or 'X' for the 10 characters.
